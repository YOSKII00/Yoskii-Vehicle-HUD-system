# 🚗 Yoskii Vehicle HUD System

🔥 Advanced all-in-one vehicle system for FiveM  
Built for **ESX servers** with full support for the modern **ox ecosystem**

---

## ⚡ Overview

Yoskii Vehicle HUD is a complete vehicle experience system combining:

- Modern NUI HUD
- Fuel system
- Keys & hotwire
- Damage & engine logic
- Mechanic wear system

All optimized into **one lightweight, high-performance resource**

---

## 🔥 Key Features

### 🖥️ Modern NUI HUD
- Clean **glass-style UI (no black background)**
- Speedometer with RPM arc
- Gear indicator (Drive, Reverse, Neutral)
- Seatbelt indicator
- Engine status + warnings
- Fuel, engine, and body bars
- Passenger minimal HUD mode
- Multiple themes:
  - Neutral
  - Dark
  - Light

---

### ⛽ Advanced Fuel System
- Built-in fuel system OR auto-detect **ox_fuel**
- Realistic fuel consumption based on RPM
- Gas station system with configurable pricing
- Petrol can support (uses ox_inventory)
- Persistent fuel saving (database)

---

### 🔑 Keys & Hotwire System
- Vehicle ownership-based keys
- Hotwire vehicles without keys
- Engine disabled without authorization
- Anti-abuse engine enforcement system

---

### 🛠️ Mechanic & Wear System
- Tracks:
  - Engine oil
  - Brakes
  - Tires
  - Suspension
  - Spark plugs
  - Air filter
  - Clutch
- Mileage-based wear system
- Fully compatible with **jg-mechanic**
- Service indicators shown in HUD

---

### ⚙️ Damage & Engine Logic
- Engine failure threshold system
- Prevents engine start if:
  - No fuel
  - Engine is too damaged
- Persistent vehicle damage tracking

---

### 🎮 Extra Features
- Seatbelt system
- Lock / unlock integration
- Mileage tracking
- Real-time vehicle data
- Notification system (ox_lib or ESX)

---

## ⚡ Compatibility

✔ **Framework:**
- ESX (`es_extended`)

✔ **Required:**
- ox_lib
- ox_inventory
- oxmysql

✔ **Optional (Auto-Detected):**
- ox_fuel (fuel system integration)
- jg-mechanic (mechanic servicing system)
